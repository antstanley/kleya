# Spec-Gap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:executing-plans`. Steps use checkbox (`- [ ]`) syntax. Repo is jj-managed; per-phase VCS sequence is documented below.

**Goal:** Close three gaps between the v0.1.0-rc.2 implementation and the design specs (see `docs/superpowers/specs/2026-05-19-spec-gaps-design.md` for the why). A fourth candidate gap (structured telemetry fields) was dropped after the parallel canonical-spec commit (`557b9000`) reconciled the wording away from the older draft's mandate.

**Architecture:** All changes are additive — no surface deprecations. Phase 1 introduces a new `CloudCompute::keypair_delete` port method (adapter and fake) and a new `LaunchService::ensure_keypair` parameter. Phase 2 is test-only.

**Tech Stack:** Rust 1.95.0, tokio (current-thread), tracing, async-trait, aws-sdk-ec2. Test runner: `cargo nextest`. VCS: jj on git backend, remote `origin` at `https://github.com/antstanley/kleya.git`.

### Step 3: Implement on the AWS adapter

In `crates/kleya-aws/src/ec2.rs`, after `keypair_fingerprint`:

```rust
async fn keypair_delete(&self, name: &KeyName) -> Result<()> {
    let res = self
        .ec2
        .delete_key_pair()
        .key_name(name.as_str())
        .send()
        .await;
    if let Err(err) = res {
        if !has_code(&err, "InvalidKeyPair.NotFound") {
            return Err(sdk(err).into());
        }
    }
    // Confirm absence — mirrors the ensure-side confirm round-trip.
    let after = self
        .ec2
        .describe_key_pairs()
        .key_names(name.as_str())
        .send()
        .await;
    if let Ok(out) = after {
        if !out.key_pairs().is_empty() {
            return Err(AwsError::MissingField("keypair still present after delete").into());
        }
    }
    Ok(())
}
```

Adjust the imports in that file only if needed — `has_code`, `sdk`, and `AwsError` are already used by neighbouring methods.

### Step 4: Update `LaunchService::ensure_keypair`

In `crates/kleya-core/src/commands/launch.rs`:

(a) Add to `LaunchOpts`:


(b) Change `ensure_keypair` signature to:

```rust
async fn ensure_keypair(&self, name: &KeyName, regenerate: bool) -> Result<()> {
```

(c) Add the new `(false, Some(_))` arm before the existing one and update the call site in `ensure_template`:

```rust
match (
    self.key_store.exists(name),
    self.compute.keypair_fingerprint(name).await?,
) {
    (true, Some(cloud_fp)) => { /* unchanged */ }
    (true, None)            => { /* unchanged */ }
    (false, Some(_)) if regenerate => {
        self.compute.keypair_delete(name).await?;
        let pair = self.key_store.generate(name)?;
        self.compute
            .ensure_default_keypair(name, &pair.public)
            .await
    }
    (false, Some(_))        => Err(Error::KeyOrphaned { name: name.clone() }),
    (false, None)           => { /* unchanged */ }
}
```

In `ensure_template`, change `self.ensure_keypair(&plan.key_name).await?;` → `self.ensure_keypair(&plan.key_name, opts.regenerate_key).await?;` — note: `ensure_template` does not currently receive `opts`. Two options:

1. Thread `regenerate_key: bool` through `LaunchPlan` (preferred; one field added to `LaunchPlan`).
2. Pass `&LaunchOpts` into `ensure_template`.

Pick **option 1** (smaller diff in the call graph). Add `pub regenerate_key: bool` to `LaunchPlan`, set it in `build_plan` from `opts.regenerate_key`, and read it from the plan in `ensure_template`.

```rust
#[tokio::test]
async fn launch_regenerates_orphaned_key_when_flag_set() {
    // Arrange: cloud has the key, local key store is empty.
    let compute = std::sync::Arc::new(InMemoryCompute::new());
    let key_store = std::sync::Arc::new(InMemoryKeyStore::new());
    let name = KeyName::new("kleya-default").unwrap();
    // Seed: register a dummy public half in the cloud only.
    compute
        .ensure_default_keypair(&name, &PublicKey("ssh-ed25519 AAAA seed".into()))
        .await
        .unwrap();
    assert!(!key_store.exists(&name)); // precondition
    assert!(compute.keypair_fingerprint(&name).await.unwrap().is_some());

    let svc = LaunchService {
        compute: compute.clone(),
        key_store: key_store.clone(),
        id_gen: std::sync::Arc::new(FakeIdGen::default()),
        config: std::sync::Arc::new(Config::default()),
    };
    let out = svc
        .run(LaunchOpts {
            template_name: None,
            instance_name: None,
            instance_type: None,
            market: None,
            dry_run: false,
            regenerate_key: true,
            cancel: None,
        })
        .await;

    out.expect("launch should succeed");
    assert!(key_store.exists(&name));
    // Fingerprint in the cloud changed (seed → fresh generated public)
    let after = compute.keypair_fingerprint(&name).await.unwrap().unwrap();
    assert!(!after.0.contains("seed"));
}
```

```rust
#![cfg(test)]

use super::ensure_floci;
use kleya_core::model::{
    instance::{InstanceFilter, InstanceId, InstanceName},
    key::{KeyName, PublicKey},
    launch::LaunchRequest,
    template::{TemplateName, TemplateSpec},
};
use kleya_core::ports::cloud_compute::CloudCompute;

fn adapter(endpoint: &str) -> kleya_aws::ec2::AwsEc2 {
    use std::sync::Arc;
    let rt = tokio::runtime::Handle::current();
    let (ec2, ssm) = rt.block_on(async {
        let ec2 = kleya_aws::client::build_ec2_client("eu-west-1", Some(endpoint)).await;
        use aws_config::BehaviorVersion;
        let cfg = aws_config::defaults(BehaviorVersion::latest())
            .endpoint_url(endpoint)
            .region(aws_sdk_ec2::config::Region::new("eu-west-1"))
            .load()
            .await;
        let ssm = aws_sdk_ssm::Client::new(&cfg);
        (ec2, ssm)
    });
    kleya_aws::ec2::AwsEc2 {
        ec2: Arc::new(ec2),
        ssm: Arc::new(ssm),
        region: "eu-west-1".into(),
    }
}

fn floci_skip(reason: &str) {
    eprintln!("floci skip: {reason}");
}

#[tokio::test]
#[ignore]
async fn instance_launch_then_list_then_terminate() {
    let Some(endpoint) = ensure_floci() else {
        floci_skip("KLEYA_TEST_FLOCI not set");
        return;
    };
    let svc = adapter(&endpoint);

    // Floci often lacks CreateLaunchTemplate; skip cleanly if so.
    let key = KeyName::new("floci-instance-test").unwrap();
    let public = PublicKey("ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIDummy floci".into());
    if let Err(e) = svc.ensure_default_keypair(&key, &public).await {
        floci_skip(&format!("ensure_default_keypair failed: {e}"));
        return;
    }
    let spec = TemplateSpec {
        name: TemplateName("floci-instance-template".into()),
        ami_id: Some(kleya_core::model::region::AmiId("ami-0123456789abcdef0".into())),
        ami_alias: None,
        instance_type: "t3.micro".into(),
        key_name: key.clone(),
        security_group_ids: vec![],
        subnet_id: None,
        market: kleya_core::model::market::MarketKind::OnDemand,
        spot_type: kleya_core::model::market::SpotType::OneTime,
        tags: vec![],
        user_data_base64: String::new(),
    };
    if svc.ensure_default_template(&spec).await.is_err() {
        floci_skip("Floci does not support CreateLaunchTemplate");
        return;
    }

    let req = LaunchRequest {
        template: spec.name.clone(),
        instance_name: InstanceName::new("floci-test").unwrap(),
        instance_type_override: None,
        market_override: None,
        spot_type_override: None,
        extra_tags: vec![],
        key_name: key,
    };
    let inst = match svc.instance_launch(&req).await {
        Ok(i) => i,
        Err(e) => { floci_skip(&format!("instance_launch failed: {e}")); return; }
    };

    let listed = svc
        .instance_list(&InstanceFilter {
            managed_only: true,
            name: Some(req.instance_name.clone()),
            states: vec![],
        })
        .await
        .expect("list");
    assert!(listed.iter().any(|i| i.id == inst.id), "launched instance must appear in list");
    assert!(listed.iter().all(|i| i.tags.iter().any(|t| t.key == "kleya:managed" && t.value == "true")));

    svc.instance_terminate(&inst.id).await.expect("terminate");
}
```

### Step 2: Bootstrap size integration test

Append to `crates/kleya-core/tests/render_snapshot.rs` (or create `crates/kleya-core/tests/bootstrap_limits.rs` if the former is snapshot-only — pick whichever already imports `render` + `encode_user_data`):

```rust
#[test]
fn render_then_encode_rejects_oversize_extra_post_lines() {
    use kleya_core::bootstrap::{encode::encode_user_data, render::{render, BootstrapVars}};
    use kleya_core::limits::USER_DATA_RAW_BYTES_MAX;

    let line = "echo padding ".repeat(64);
    let extras: Vec<String> = (0..96).map(|_| line.clone()).collect();
    let total: usize = extras.iter().map(String::len).sum();
    assert!(total > USER_DATA_RAW_BYTES_MAX * 4, "precondition: total={total}");

    let vars = BootstrapVars {
        install_ghostty_terminfo: false,
        ghostty_terminfo_source: "",
        install_dev_tools: true,
        node_major: 22,
        python_version: "3.12",
        extra_pre_lines: &[],
        extra_post_lines: &extras,
    };
    let rendered = render(&vars).expect("render");
    let err = encode_user_data(&rendered).expect_err("must reject");
    assert!(
        matches!(err, kleya_core::Error::UserDataTooLarge { .. }),
        "got: {err:?}",
    );
}
```

Adjust the `BootstrapVars` literal to match the actual struct in `crates/kleya-core/src/bootstrap/render.rs` — read it first and copy field names exactly. Many of these fields may have defaults via `BootstrapVars::default_with(...)`; prefer the existing constructor and only override `extra_post_lines`.
